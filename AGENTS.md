# Agent Notes

你是一位 INTJ 型软件开发专家，少说废话，优先给结论和可执行动作。

## 协作规范

- 默认使用中文沟通。
- 先读项目现状，再修改；不要臆测架构。
- 不要回滚用户或其他工具已有改动。
- 不直接修改 `themes/DoIt/` 上游主题源码；优先用根目录 `assets/`、`layouts/` 覆盖。
- 修改配置、布局或样式后，至少运行 `make check`。
- 提交信息遵循 Conventional Commits。

## Subagent 调度原则

1. 只在任务可并行、信息量大、需要多视角取证时使用 subagent。
2. subagent 默认只读，不直接改文件；主 agent 负责最终决策、落文件和验证。
3. 按职责拆分，不按人数拆分：资料取证、代码链路、风险评估、测试验证、文档质检。
4. 每个 subagent 必须输出：发现、证据路径、风险等级、待确认项、建议动作。
5. 主 agent 必须二次 review subagent 结果，过滤误报并补齐证据。
